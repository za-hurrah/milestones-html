Basic Tags

Text

Block level:

  • The p tag allows for text to appear. The desried text should go inside the two p brackets.
  • The blockquote tag allows shows citation from another website. Here is an example:
  • The pre tag preserves the formatting of the text just as it is entered in the code. The text should be within the brackets, just like the p tag.

Inline styling

  • The em tag can be used to emphasize certain parts of the text. This shows up as italics in most softwares.
  • The i tag formats the text in italics.
  • Likewise, the strong tag makes ceratin parts of the text stronger than the rest. This shows up as bold in most softwares.
  • The b tag formats the text as bold.
    • b is a style - we know what "bold" is supposed to look like.
    strong however is an indication of how something should be understood. "Strong" could (and often does) mean "bold" in a browser, but it could also mean a lower tone for a speaking program like Jaws (for blind people) or be represented by an underline (since you can't bold a bold) on a Palm Pilot.
  • The code tag defines a line of code in the default monospace font.

Additional tags

  • The a tag defines a hyperlink. Specific info about references in this tag can be found on the anatomy page

Tables

Tables have unique tags to define content:

  • The table tag defines the code as a table
  • Table rows are defined by the tr tag
    • To add more rows, just list as many tr tags as you would like
  • There are two standard tags for table content:
    1. The th tag defines the table header, which is automatically bold and centered
    2. The td tag defines the table data or content, which is regular font and left aligned
