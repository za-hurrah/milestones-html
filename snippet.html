Here, the tag 'p' defines the content of a paragraph. The text after the first p bracket and before the /p bracket will appear on the user interface. This site provides more detail about the p tag.
The tag 'a' defines a hyperlink. Inside the first bracket are where you'll add internal labels about the hyperlink. The text between the two a brackets are what will be hyperlinked. This site provides more detail about the a tag.

The tags are nested within in each other. The information within the a tag only applies to that section. Anything done within the p tag applies to all of it, including the a tag. The a tag would be considered a child of the p tag, and the p tag the parent of the a tag. This lineage can continue; there can be multiple children of one parent, a child of a child, and so on.

Labels can be added to any part of the code to identify and reference later on. In this example, there are labels for the hyperlink. Specifically:

  • 'href' provides the link
  • 'id' specifies that line of code
  • 'class' categorizes it as a banana, there may other members of the banana class elsewhere in the code
  • 'target' describes how the command should be operated. Here _blank calls for it to be opened in a blank tab
